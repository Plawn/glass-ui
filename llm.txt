# glass-ui-solid

> iOS 26-inspired glassmorphism UI components for SolidJS with Tailwind CSS v4.

Package: `glass-ui-solid` v0.4.2
License: Apache-2.0
Peer dependency: `solid-js` ^1.8.0
Repository: https://github.com/Plawn/glass-ui

## Installation

```bash
npm install glass-ui-solid solid-js
```

## Setup

Import global styles in your app entry point:

```tsx
import 'glass-ui-solid/styles.css';
```

For Tailwind CSS v4 integration, import the theme source:

```css
@import 'glass-ui-solid/theme.css';
```

Dark mode uses class-based toggling (add `.dark` class to a parent element).

## Import Pattern

All components and hooks are exported from the package root:

```tsx
import { Button, Modal, toast, useDisclosure } from 'glass-ui-solid';
```

## Components

### Button

Interactive button with variants and loading state.

```tsx
<Button variant="primary" size="md" loading={false} onClick={() => {}}>
  Click me
</Button>
```

Props:
- `variant?: 'primary' | 'secondary' | 'tertiary' | 'ghost' | 'danger'` (default: `'primary'`)
- `size?: 'sm' | 'md' | 'lg'` (default: `'md'`)
- `type?: 'button' | 'submit' | 'reset'` (default: `'button'`)
- `fullWidth?: boolean`
- `disabled?: boolean`
- `loading?: boolean` — shows spinner
- `leftIcon?: JSX.Element`
- `rightIcon?: JSX.Element`
- `class?: string`

### Input

Text input with label, icons, and error.

```tsx
const [value, setValue] = createSignal('');
<Input value={value()} onInput={setValue} label="Email" type="email" placeholder="you@example.com" error="Required" />
```

Props:
- `value: string` (required)
- `onInput: (value: string) => void` (required)
- `type?: 'text' | 'password' | 'email' | 'number' | 'url' | 'tel' | 'search'` (default: `'text'`)
- `label?: string`
- `placeholder?: string`
- `size?: 'sm' | 'md' | 'lg'` (default: `'md'`)
- `error?: string`
- `disabled?: boolean`
- `readonly?: boolean`
- `required?: boolean`
- `id?: string`, `name?: string`, `autocomplete?: string`
- `leftIcon?: JSX.Element`, `rightIcon?: JSX.Element`
- `class?: string`

### Select

Dropdown select. Supports two modes: children-based (native options) or options array (typed).

```tsx
// Children mode
<Select value={val()} onChange={setVal} label="Country">
  <option value="us">United States</option>
  <option value="fr">France</option>
</Select>

// Options array mode (typed)
<Select
  value={selected()}
  onChange={setSelected}
  options={[{ value: 'us', label: 'United States' }, { value: 'fr', label: 'France' }]}
  emptyOption="Select a country"
  label="Country"
/>
```

Props:
- `value: string | T | null` (required)
- `onChange: (value: string | T | null) => void` (required)
- `children?: JSX.Element` — native option elements
- `options?: { value: T; label: string; disabled?: boolean }[]`
- `emptyOption?: string` — placeholder when no selection
- `label?: string`, `size?: 'sm' | 'md' | 'lg'`, `error?: string`, `disabled?: boolean`

### Toast (Global Notification)

Temporary notification messages. Requires `<ToastContainer />` in app root.

```tsx
// In app root
<ToastContainer />

// Anywhere in code
import { toast, dismissToast, clearToasts } from 'glass-ui-solid';
toast.success('Item saved!');
toast.error('Something went wrong');
toast.warning('Low disk space');
toast.info('Update available');
// Optional duration: toast.success('Saved', 5000);
```

### Snackbar (Global Notification)

Bottom notification with optional action. Requires `<SnackbarContainer />` in app root.

```tsx
// In app root
<SnackbarContainer />

// Anywhere in code
import { showSnackbar, dismissSnackbar, clearSnackbars } from 'glass-ui-solid';
showSnackbar({ message: 'Item deleted', action: 'Undo', onAction: handleUndo, duration: 4000 });
```

Options:
- `message: string` (required)
- `action?: string`, `onAction?: () => void`
- `duration?: number` (default: `4000`, `0` to persist)
- `position?: 'bottom-left' | 'bottom-center' | 'bottom-right'` (default: `'bottom-center'`)

### ContextMenu

- `ContextMenuItem` props: `onSelect?: () => void`, `disabled?: boolean`, `icon?: JSX.Element`, `shortcut?: string`, `destructive?: boolean`

## Theming

The library uses Tailwind CSS v4 with OKLCH color space. Colors are defined as CSS custom properties on `:root` and `.dark`.

Color scales available: `surface` (50-950), `accent` (50-900), `error` (50-900).

Glass effects use `backdrop-filter: blur()` with semi-transparent backgrounds for the frosted glass look.

To customize the theme, override CSS custom properties:

```css
:root {
  --color-accent-500: oklch(0.6 0.2 250);
}
```

## Accessibility

All components follow WAI-ARIA patterns:
- Focus trapping in overlays (Modal, Drawer, Dialog, CommandPalette)
- `aria-invalid` and `aria-describedby` on form inputs
- Keyboard navigation (arrow keys in RadioGroup, Tabs, CommandPalette)
- `aria-activedescendant` in Autocomplete and CommandPalette
- Screen reader support with proper roles and labels
